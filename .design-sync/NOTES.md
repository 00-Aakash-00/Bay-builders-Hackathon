# design-sync notes тАФ CustomerZero

This repo is a **Next.js app, not a packaged design system**, so this is an
**off-script** package-shape sync. Read this before any re-sync.

## How the off-script build is wired
- **No dist, no synth-entry.** An explicit re-export entry names the exact
  components: `.design-sync/assets/ds-entry.tsx`. The synth-entry fallback is
  deliberately avoided тАФ it `export *`s every `src/` file and would drag
  `next/*` + server code into esbuild and fail. Always build with
  `--entry .design-sync/assets/ds-entry.tsx`.
- **7 components:** IcpPicker, LeadCard (app), CandidateTracker, RadarFeed,
  SwarmFeed (group `app`); DitherPhoto, ReceiptCard (group `marketing`).
- **LeadCard collision:** app `LeadCard` and marketing `LeadCard` both export
  `LeadCard`. The marketing one is re-exported as **ReceiptCard** in ds-entry.
  Because its source export name (`LeadCard`) тЙа component name (`ReceiptCard`),
  its props are hand-written in `cfg.dtsPropsFor.ReceiptCard` тАФ keep them in
  sync with `src/components/marketing/lead-card.tsx` if that changes.
- **next/dynamic shim:** DitherPhoto imports `next/dynamic`. It's remapped to
  `.design-sync/assets/next-dynamic-shim.tsx` via `cfg.tsconfig` тЖТ
  `.design-sync/tsconfig.ds.json` (`paths`). That tsconfig also carries `@/*`.
  It's the ONLY `next/*` import across the 7 components.

## Styles (the big fidelity dependency)
- Tailwind v4 utilities exist nowhere as static CSS in this repo (Next compiles
  them at build). They are compiled OFFLINE into `cfg.cssEntry`:
  `.design-sync/assets/ds-styles.css`.
- **Recompile whenever component source changes classes, or tokens change:**
  ```
  npx @tailwindcss/cli@4 -i .design-sync/assets/tw-input.css -o .design-sync/assets/compiled.css
  # then prepend the Google-fonts @import back:
  printf '@import url("https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&family=Geist+Mono:wght@400;500;600&display=swap");\n' | cat - .design-sync/assets/compiled.css > .design-sync/assets/ds-styles.css
  ```
  `tw-input.css` `@source`s the 7 component files and copies the `@theme` from
  `src/app/globals.css` verbatim. If `globals.css` tokens change, update
  `tw-input.css` too.
- CSS-module rules (`app-motion.module.css`, `marketing.module.css`) are handled
  automatically by esbuild's local-css loader тЖТ `_ds_bundle.css`.

## Fonts
- Inter + Geist Mono load via a remote `@import` (prepended to ds-styles.css) тЖТ
  `[FONT_REMOTE]`, non-blocking.
- **Fraktion** is referenced by the theme but never shipped by the app (it falls
  back to system sans in production too) тЖТ suppressed via
  `cfg.runtimeFontPrefixes: ["Fraktion"]`. Do not treat `[FONT_MISSING]` for
  Fraktion as a bug.

## Dithered brand images (DitherPhoto)
- `DitherPhoto` renders a live WebGL ordered-dither. Its preview points at
  pre-baked images in `.design-sync/assets/brand-dithered/` (proof, web, swarm,
  signal), generated by `.design-sync/.imgtool/dither.mjs` (sharp; 4x4 Bayer,
  colorSteps=2 тЖТ a 4-color palette #0b0d12/#6c7185/#cdd4f7/#ffffff; `signal`
  uses inverted=false). Re-run that script if `public/brand/*.webp` change.

## Known render warns
- `[RENDER_SKIPPED]` on every validate (expected тАФ `--no-render-check`).
- `[FONT_REMOTE]` Inter/Geist Mono (expected).

## Re-sync risks (watch-list)
- **Stale styles:** if a component's className strings change and ds-styles.css
  isn't recompiled, the new utilities render unstyled. Recompile per above.
- **Hand-maintained surfaces:** `ds-entry.tsx`, `cfg.componentSrcMap`, and
  `cfg.dtsPropsFor` are manual. Adding/removing a component means editing
  ds-entry.tsx AND componentSrcMap (and dtsPropsFor if an aliased export).
- **guidelines/** shipped the repo's internal `docs/PRD.md` +
  `docs/architecture.md` (default `guidelinesGlob`). Set `cfg.guidelinesGlob` to
  narrow/exclude if those shouldn't be in the DS.
- **CandidateTracker** uses `<style jsx global>` (styled-jsx). Outside Next it
  renders as a plain `<style>` (CSS still applies; benign React attr warning).
- Grades/renders are not machine-anchored тАФ carry-forward skips won't apply the
  way they do for a verified sync; expect a re-sync to re-verify.
